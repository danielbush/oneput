# chore: Type checks for tests

## Problem

`task check` only type-checks production code in `packages/jsed/`. Test files (`src/__tests__/**/*.test.ts`, `src/lib/__tests__/**/*.test.ts`) are excluded. The editor's built-in type checker sometimes flags errors in tests that `task check` misses, which is confusing.

## Goal

Determine whether we can include test files in type checking, and if so, enable it. If it's not feasible or introduces too many issues, document why and close the ticket.

## How to do this

1. Read `packages/jsed/CLAUDE.md` and the `/jsed` skill to understand the codebase structure.

2. **Investigate current config** — look at the tsconfig files in `packages/jsed/`:
   - Find which tsconfig `task check` uses (check `Taskfile.yml` at the repo root and `packages/jsed/` level)
   - Identify what excludes test files — is it an `exclude` pattern, a separate tsconfig, or something else?
   - Check if there's already a tsconfig that includes tests (e.g. `tsconfig.test.json`)

3. **Try enabling it** — the simplest approach is usually adding test file globs to the existing tsconfig's `include`, or removing them from `exclude`. Try this and run `task check` to see what happens.

4. **If there are type errors in tests:**
   - Count them and categorise them (missing types, wrong argument types, import issues, etc.)
   - If there are fewer than ~10 and they're straightforward, fix them
   - If there are many or they require significant rework, document the categories and counts, then propose whether it's worth fixing now or later

5. **If it works cleanly**, update the config so `task check` includes tests going forward.

## Verification

```bash
task check   # should now also check test files
task test    # all tests still pass
```

## Scope

- Only touch `packages/jsed/` config and test files
- Do not change any production code
- Do not change test logic — only fix type annotations/imports if needed

## Changes

- **2026-03-24** — removed `"exclude": ["**/__tests__"]` from `tsconfig.json`; fixed 10 errors across 4 test files: unused imports (`it`, `isCollapsed`, `isBlockTransparent`) and obsolete `TokenManager` constructor argument (7 occurrences). All tests pass.
