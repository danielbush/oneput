# refactor: isToken type guard

## Problem

`isToken` in `packages/jsed/src/lib/traversal.ts` is currently a type guard:

```ts
export function isToken(
  el: EventTarget | Element | Node | ChildNode | ParentNode | null | undefined
): el is HTMLElement {
```

This narrows the type to `HTMLElement` when `isToken` returns true. But now that LINE_SIBLING traversal handles non-TOKEN elements (ISLANDs, CURSOR_BOUNDARYs, TRANSPARENT_BLOCK) ŌĆö which are also `HTMLElement`s ŌĆö the type guard doesn't actually distinguish TOKENs from other elements at the type level. It gives false confidence: code after `if (isToken(el))` looks like it's working with a TOKEN, but the narrowed type is just `HTMLElement`, which is the same type as an ISLAND.

The question is whether this type guard is misleading or whether it's still useful for narrowing from broader types like `Node | null`.
