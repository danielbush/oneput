# feat: click/touch tokenizes LINE and enters edit mode

## Problem

Click or touch on a LINE should tokenize it and put the CURSOR at the beginning of the first TOKEN. Currently clicking a LINE focuses it but doesn't enter edit mode — the user has to explicitly trigger edit mode separately.

## Design

"Focus twice to edit" model: the first click/touch on a FOCUSABLE sets FOCUS (view mode — lets the user click around the document structure without entering edit mode). A second click/touch on the *same* FOCUSABLE triggers quick-descend to find the first TOKEN and enter edit mode. This keeps navigation and editing clearly separated.

When REQUEST_FOCUS targets the already-focused element:
1. Quick-descend into it (see QUICK_DESCEND__WORK)
2. If a TOKEN is found, enter edit mode with CURSOR on it
3. If quick-descend returns null (no editable content), stay in view mode

Previous work already arranged for the CURSOR to be removed when the user clicks away from the CURSOR_LINE, so exiting edit mode is handled. This model may evolve but is the starting point.

Click or touch on a LINE should tokenize it and put the CURSOR at the beginning of the first TOKEN. If FOCUS is already on that LINE (e.g. second tap), quick-descend into it — find the first text/TOKEN, establish the LINE, and enter edit mode directly. See QUICK_DESCEND__WORK.
