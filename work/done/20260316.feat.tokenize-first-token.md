## Changes

### tokenizeLine returns first TOKEN

- `tokenizeLine` returns the first TOKEN created during SHALLOW_TOKENIZATION
- This lets callers know where the CURSOR should land without a separate lookup step

### Vocabulary

- **SHALLOW_TOKENIZATION** — tokenization scoped to a single LINE, not recursing into NESTED_LINE's. Driven by browser performance: each DOM node insertion triggers layout/paint/memory pressure in large documents
- **LINE_MEMBER** — umbrella for everything inside a LINE (TOKEN, INLINE, NESTED_LINE, ISLAND)
- **LINE_SIBLING** narrowed to TOKEN's only — CURSOR traverses LINE_SIBLING's, so NESTED_LINE's and ISLAND's are excluded

### TokenManager refactor

- `tokenize` uses `tokenizeLine` return value directly — no more `getNextLineSibling` lookup after tokenization
- Migrated from walk.js to walk2.js (`findNextNode`); descends into child FOCUSABLE's to find the first LINE with text

## Decisions

- **Return value over mutable ref** — first TOKEN threaded through recursion via return value, not a shared `{ token }` object. Each level returns first or null; symmetric across both branches
- **SHALLOW_TOKENIZATION is about the browser, not tokenization time** — DOM mutations are expensive (layout, paint, memory); tokenizing the whole JsedDocument eagerly would cause jank
- **LINE_SIBLING = TOKEN's only** — CURSOR should not visit NESTED_LINE's or ISLAND's as siblings; they're LINE_MEMBER's but not traversable in the same sense
- **Inline-block `<span>` is a NESTED_LINE, not an ISLAND** — it's a FOCUSABLE (not INLINE) with a LINE ancestor; ISLAND has different semantics
- **`<p>` can't contain `<div>`** — browser auto-closes the `<p>`, so tests use `<div>` wrappers for nesting
- **walk2 param inversion** — `ignoreDescendents: isIsland` becomes `descend: (node) => !isIsland(node)`
