# refactor: Flip OPAQUE_BLOCK to the default

See OPAQUE_BLOCK_DEFAULT__WORK for the full discussion and motivation.

Continues from CURSOR_WALKS_NON_TOKENS__WORK which established the CURSOR_OPAQUE / CURSOR_TRANSPARENT taxonomy and introduced the `jsed-cursor-opaque` class. This ticket inverts the default: non-INLINE, non-ISLAND FOCUSABLE's are now OPAQUE_BLOCK (CURSOR visit=yes, descend=no) unless explicitly marked `jsed-cursor-transparent`.

## Plan

### 1. Constants (`src/lib/constants.ts`)

- Add `JSED_CURSOR_TRANSPARENT_CLASS = 'jsed-cursor-transparent'`
- Remove `JSED_CURSOR_OPAQUE_CLASS`

### 2. Predicates (`src/lib/traversal.ts`)

- **`isCursorBoundary`** — flip: a LINE is a cursor boundary unless it has `jsed-cursor-transparent` class. Rename to reflect the inversion (or keep the name — it still means "is this element a boundary the CURSOR won't cross?")
- **`isTransparentBlock`** — becomes `isLine(el) && el.classList.contains(JSED_CURSOR_TRANSPARENT_CLASS)`
- **`isLineSibling`** — currently `isToken || isIsland || isCursorBoundary`. After the flip, a LINE that is *not* transparent is a LINE_SIBLING: `isToken || isIsland || (isLine(el) && !isTransparentBlock(el))`

### 3. Tokenization (`src/lib/token.ts`)

No logic change — `tokenizeLineRec` gates on `isTransparentBlock(child)` (line 170 calls `isBlockTransparent` which is `isTransparentBlock`). The flipped predicate narrows tokenization automatically.

### 4. Tests

- lineSibling and TokenCursor tests that use `jsed-cursor-opaque` flip to `jsed-cursor-transparent` on elements that should be transparent
- Tests relying on the default being transparent need adjusting — nested blocks in test fixtures are now opaque by default
- Verify SHALLOW_TOKENIZATION: tokenizing a LINE with nested blocks should not recurse into them

### 5. Docs

- `vocabulary.md` — swap which is the default in OPAQUE_BLOCK / TRANSPARENT_BLOCK descriptions
- `architecture.md` — minor wording update

### Not affected

- **Nav** — doesn't know about opaque/transparent, only sees FOCUSABLE's
- **InputManager / EditManager** — delegate to TokenCursor which delegates to traversal predicates
- **`tokenizeLine`** — delegates to `isTransparentBlock`, follows automatically
