# fix: Stale CURSOR_LINE on click-to-focus

Extracted from CURSOR_WALKS_NON_TOKENS__WORK.

## Problem

When editing a TOKEN and then clicking a TOKEN in a different LINE, `moveNext` fires with the old CURSOR_LINE as ceiling. This produces the warning: `findNextNode: start [HTMLSpanElement] is not contained in ceiling [HTMLDivElement]`.

The CURSOR_LINE stored on TokenCursorBase becomes stale when REQUEST_FOCUS hops across LINE boundaries.

## Possible approaches

1. Update CURSOR_LINE when the CURSOR hops to a TOKEN in a different LINE
2. Tear down the old edit session before the click triggers `moveNext`, then re-establish the LINE ceiling on REQUEST_FOCUS
