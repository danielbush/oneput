# fix: PADDED_TOKEN should apply after block elements

## Problem

`tokenizeLine` only pads the first TOKEN after an ISLAND (via `isIsland` check in `replaceTextNode`). But any LINE element (block, inline-block, etc.) also doesn't carry trailing space to the next sibling text node. The TOKEN after them loses its leading space.

Example: `<div>aaa <span style="display:inline-block">inner</span> bbb</div>` — the TOKEN "bbb" should be padded because the inline-block span doesn't provide trailing space, same as an ISLAND.

## Solution

The right predicate is `isIsland(prev) || isLine(prev)` — pads after any block or ISLAND, since neither carries trailing space to the next sibling text node. TRANSPARENT_BLOCKs don't actually trigger this path because tokenization descends into them and absorbs trailing text. The `remove` padding transfer logic is already generic (`isPadded` check) and needs no change.
