# feat: REC_NEXT/SIB_NEXT and REC_PREV/SIB_PREV close CURSOR and move FOCUS

## Problem

When the CURSOR is open (editing a LINE), pressing REC_NEXT/SIB_NEXT or REC_PREV/SIB_PREV does nothing useful. The user has to explicitly exit edit mode before navigating. This adds friction to the editing flow.

## Design

When the CURSOR is active and a navigation action (REC_NEXT, SIB_NEXT, REC_PREV, SIB_PREV) is triggered:
1. Close the CURSOR (de-tokenize, return to view mode)
2. Move FOCUS to the appropriate FOCUSABLE using the corresponding Nav action

This makes navigation seamless — the user can move through the document without manually toggling modes.
