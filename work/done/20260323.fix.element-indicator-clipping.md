# fix: ElementIndicator clipped when element is near left edge

## Problem

The ElementIndicator badge (shows the tag name like "P", "DIV" near the focused element) positions itself with its right edge aligned to the right edge of the focused element, using `translateX(-100%)`. When the focused element is small and near the left edge of its container, the badge overflows to the left and gets clipped.

## Goal

Detect when right-aligned positioning would clip the badge, and fall back to left-edge alignment (badge's left edge aligned with the element's left edge).

## How to do this

1. Read `packages/jsed/CLAUDE.md` and the `/jsed` skill to understand the codebase structure.

2. **Read the current implementation** — the positioning logic is in `ElementIndicator.#makeIndicator` in `packages/jsed/src/ElementIndicator.ts`. Currently it:
   - Gets the element's `getBoundingClientRect()`
   - Positions the badge at `left: rect.right` with `translateX(-100%)` (right-aligned to element)
   - Already handles vertical clipping: if there's not enough space above, it positions below instead

3. **Add horizontal clipping detection** — follow the same pattern as the existing vertical check:
   - After creating the badge and measuring it (the code already does a measure pass for height), also measure the badge's width
   - Calculate where the badge's left edge would end up: `rect.right - badgeWidth`
   - If that value is negative (off the left edge of the viewport), switch to left-aligned: `left: rect.left` without the `translateX(-100%)`
   - This gives you 4 possible positions: above-right (default), below-right, above-left, below-left

4. **Implementation notes:**
   - The badge width can be measured the same way the height is — the code already temporarily appends the badge with `visibility: hidden` to measure `offsetHeight`. Just also read `offsetWidth` at the same time.
   - Keep the existing vertical logic intact — just add the horizontal check alongside it
   - The badge uses `position: fixed`, so clipping is relative to the viewport, not a scroll container

5. **Test manually** using `apps/jsed-demo/`:
   - `task dev` to start the demo
   - Focus on elements near the left edge of the page
   - Focus on small elements (like a short `<em>` or single-word `<span>`)
   - Verify the badge is fully visible in all positions
   - Verify the badge still positions correctly for elements in the middle/right of the page (no regression)

6. **Write a unit test** using `ElementIndicator.createNull()` if feasible. The null version uses `NullElement` which returns `offsetHeight = 0`, so you may need to make `NullElement` configurable for this. If testing is awkward, manual testing is sufficient — this is a small visual fix.
