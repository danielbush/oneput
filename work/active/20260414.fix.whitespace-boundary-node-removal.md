# fix: whitespace boundary node removal

## Situation

The current whitespace-removal behavior is in transition. Jsed vocabulary now defines `LEADING_SPACE` and `TRAILING_SPACE` as whole whitespace text nodes at structural boundaries, but some low-level helpers and tests were written around trimming one leading or trailing whitespace character from an intervening text node instead.

Recent changes renamed the helpers to `removeLeadingSpaceNode(...)` / `removeTrailingSpaceNode(...)` and changed them to remove the whole whitespace node. Most tests still pass, but a small set of tag-side tests still reflect the older partial-trim behavior. There is also a broader question about how cursor-side and tag-side whitespace operations should align with the vocabulary and with real app behavior.
