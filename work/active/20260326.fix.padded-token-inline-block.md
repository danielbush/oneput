# fix: PADDED_TOKEN should apply after inline-block elements

## Problem

`tokenizeLine` only pads the first TOKEN after an ISLAND (via `isIsland` check in `replaceTextNode`). But inline-block elements (whether OPAQUE_BLOCK or TRANSPARENT_BLOCK) also don't carry trailing whitespace, so the TOKEN after them loses its leading space.

Example: `<div>aaa <span style="display:inline-block">inner</span> bbb</div>` — the TOKEN "bbb" should be padded because the inline-block span doesn't provide trailing space, same as an ISLAND.

## Where

`packages/jsed/src/lib/token.ts` — `replaceTextNode` around line 132. The condition checks `isIsland(getPreviousVisibleSibling(...))` but should also check for other CURSOR_OPAQUE elements that don't carry trailing space — specifically any non-INLINE LINE (inline-block, etc.).
